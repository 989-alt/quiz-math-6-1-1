# 난이도 3단 모드 설계 (2026-07-13)

사용자 피드백: "게임 난이도가 낮아 문제를 풀어야 할 당위성이 떨어진다. 몬스터 수·이동 속도 등으로 난이도를 높일 수 있는가."

진단: 당위성 = ① 게임이 충분히 위협적인가(업그레이드가 생존 필수인가) × ② 퀴즈 결과가 위협에 직결되는가.
현재는 몬스터 최고속 105 vs 플레이어 200으로 무한 카이팅이 가능하고(①), 오답 페널티가 "레벨업 소모"뿐(②)이라 둘 다 약함.

## 확정 방향 (사용자 선택)

- 난이도 3단 선택 모드(쉬움/보통/어려움), 시작 화면에서 선택, 모드별 차이 명시
- 시간 압박(생존 시간 기반 자동 가속)은 **전 모드** 적용, 강도만 차등
- 기본 난이도 상향은 보통/어려움에 차등 적용
- 오답 페널티는 **어려움 전용**, 3종 중첩(체력 감소 + 광폭화 + 미니 러시)
- 랭킹은 **난이도별 별도**(필드+탭 분리, 컬렉션 유지)

## 1. 아키텍처

- `src/game/difficulty.ts` 신설 = 단일 소스. `Difficulty = 'easy' | 'normal' | 'hard'`,
  `DIFFICULTY_CONFIG`에 statMul(hp/speed/damage), spawnIntervalMul, maxActive, timeRamp(perMinute/cap), wrongPenalty, label, descriptions.
- 적용 지점 3곳: ① Monster config 생성(getMonsterConfigForRotation/getBossConfigForWave) ② GameScene.updateMonsterSpawning(간격·상한) ③ GameScene.handleQuizResult(어려움 오답 페널티).
- 전달: StartScreen → onStart(nickname, difficulty) → Phaser registry → GameScene 시작 시 1회 읽기. 게임 중 변경 불가.
- **기존 밸런스 = 쉬움**(배율 1.0 + 약한 시간 가속만). XP 경제·클리어 조건 무수정.

## 2. 난이도별 수치

| 항목 | 쉬움 | 보통 | 어려움 |
|---|---|---|---|
| 몬스터 HP | ×1.0 | ×1.2 | ×1.4 |
| 이동속도 | ×1.0 (60~105) | ×1.15 (~121) | ×1.3 (~137) |
| 공격력 | ×1.0 | ×1.1 | ×1.25 |
| 스폰 간격 | ×1.0 | ×0.8 | ×0.65 |
| 동시 상한 | 100 | 120 | 140 |
| 시간 가속(분당) | 속도+2%/HP+3% | 속도+3%/HP+5% | 속도+4%/HP+7% |
| 가속 상한 | +20% | +35% | +50% |
| 오답 페널티 | — | — | 발동 |

- 속도가 난이도의 몸통: 어려움 후반 몬스터 최고 ~205로 플레이어(200)와 대등 → 카이팅 파산, 화력(=퀴즈 정답) 필수화.
- 시간 가속은 처치수 기반 웨이브의 카이팅 허점(안 싸우면 난이도 정지)을 막는 별도 축.
- HP는 소폭만: 과도하면 처치↓→XP↓→퀴즈 빈도↓로 교육 목적 훼손.

## 3. 어려움 오답 페널티 (3종 중첩, 개별 강도는 절반 수준)

1. **체력 -10%(즉시)** — 최대 체력 기준, 체력 1 미만 불가(오답 단독 사망 없음). 피격 플래시 + "-10 HP" 표시.
2. **광폭화(재개 직후 10초)** — 전체 몬스터 속도 +25%, 빨간 틴트 + 러시 비네트 재사용. 연속 오답 시 연장 아닌 리셋.
3. **미니 러시(재개 보호 종료 후)** — 기존 러시 스포너 축소판: 3초간 ~10마리. WARNING 없이, 단 재개 보호(3·2·1+밀쳐내기+1.5s 무적) 이후 시작. 진행 중 재발동 금지.

- 재개 보호는 오답 시에도 항상 적용. 미니 러시 몬스터도 XP 젬 드랍(위기=만회 기회).
- 정답 보상은 전 난이도 동일(별도 랭킹이므로 점수 인플레 무관).

## 4. UI · 랭킹

- StartScreen 닉네임 아래 난이도 카드 3장(기본=보통), 카드에 차이점 명시(어려움 카드에 오답 페널티 3종 명시). HUD에 난이도 배지 상시 표시.
- `ScoreEntry.difficulty` 필드 추가, 경로 `leaderboards/{unitId}/scores` 유지, 리더보드에 쉬움/보통/어려움 탭(클라이언트 필터). 구기록(필드 없음)= '쉬움' 탭(기존 밸런스=쉬움이므로 의미 정확).
- Firestore rules에 `difficulty in ['easy','normal','hard']` 검증 추가 → 콘솔 재게시(사용자 액션).

## 5. 검증

1. `npm run build` green
2. Playwright 라이브: 난이도별 속도/스폰 실측, 어려움 오답 3종 발동·체력 1 방어·중첩 방지
3. 랭킹 3난이도 제출 → 탭 필터·구기록 쉬움 분류 확인
