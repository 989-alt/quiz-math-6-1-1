## 0. 배경과 제약

- **제출처**: issamGPT AI Mart 바이브코딩 콘텐츠 제작 프로젝트 (기한 2026-11-30)
- **등록 콘셉트**(시트 원문 요지): 뱀서라이크 게이미피케이션. 초5-6 수학/사회/국어 개념 퀴즈로
  캐릭터 강화. 방향키 이동, 몬스터 처치, 수정 획득 → 포인트/경험치, **레벨업 시 퀴즈 →
  정답이면 무기 강화**. → 이 코어 루프는 반드시 유지.
- **런타임 LLM API 사용 금지**. 문제는 정적 문제은행으로 제공.
- **랭킹**: Firebase Firestore 유지 + 접속 실패/차단 시 localStorage 폴백 (사용자 확정).
- **아트 스타일 = 픽셀아트 확정** (키비주얼 A/B 비교 후 사용자 결정, §8 참고).
- 대상 기기: 학교 PC/크롬북/태블릿 (키보드 + 터치 조이스틱).
- 이미지 생성: **kie.ai gpt-image-2** (키 `.env.kie`, 커밋 금지, 비용 제약 없음).

## 1. 문제은행 — 20단원 × 300문제 = 6,000문제

### 1.1 범위 (issamGPT 시트 "이선학 선생님" 연관 차시 기준, 사용자 확정)

| 과목 | 단원 |
|------|------|
| 수학 5-1 | ①자연수의 혼합 계산 ②약수와 배수 ③대응 관계(규칙과 대응) ④약분과 통분 ⑤분수의 덧셈과 뺄셈 |
| 수학 5-2 | ②분수의 곱셈 ④소수의 곱셈 ⑥평균과 가능성 |
| 수학 6-1 | ①분수의 나눗셈 ③소수의 나눗셈 ④비와 비율 |
| 수학 6-2 | ①분수의 나눗셈 ②소수의 나눗셈 ④비례식과 비례배분 ⑤원의 넓이 |
| 사회 5-2 | ①새로운 사회를 향한 움직임 ②일제의 침략과 광복을 위한 노력 ③대한민국 정부의 수립과 6.25 전쟁 |
| 국어 6-1 | 바르게 고쳐 써요(맞춤법·문법) / 상황에 맞게 표현해요(관용 표현) |

수학 15 + 사회 3 + 국어 2 = 20단원. 현행 "수학 24단원" 데이터는 이 20개로 교체.

### 1.2 데이터 구조

```
src/data/banks/
  math/g5-1-1.json ... g6-2-5.json   (15개)
  social/g5-2-s1.json ... s3.json    (3개)
  korean/g6-1-k1.json, k2.json       (2개)
src/data/banks/index.ts              (메타 + dynamic import 매핑)
```

- 한국어 문항은 **JSON 파일**로 분리 (CLAUDE.md 규칙: TS/JS 인라인 한국어 금지).
- 단원당 300문항 ≈ 60~90KB → 선택한 단원만 `import()` lazy load.
- 스키마 확장:

```ts
interface BankQuiz {
  id: string;
  type: 'calc' | 'concept' | 'word' | 'fact' | 'order' | 'idiom' | 'spelling';
  difficulty: 1 | 2 | 3;          // 하/중/상
  question: string;
  options: [string, string, string, string];
  correctIndex: 0 | 1 | 2 | 3;
  explanation: string;            // 필수화 (오답 학습 피드백)
}
```

### 1.3 조달 방식 (전부 빌드 타임, 런타임 API 없음)

- **수학 계산형(단원당 ~180문)**: 결정적 템플릿 생성기(`scripts/bankgen/`).
  파라미터 랜덤 + 시드 고정, 정답 자동 산출, 오답지는 "그럴듯한 실수" 규칙으로 생성
  (연산 순서 무시값, 자릿수 실수값, 근접값). 생성 결과를 JSON으로 구움.
- **수학 개념/문장제(단원당 ~120문)**: 정제 작성 (교육과정 성취기준 기반).
- **사회 3단원(각 300문)**: 한국사 사실 기반 정제 작성. 유형 배분 — 인물/사건/연대순(order)/
  인과/키워드. 민감 표현 검수(전쟁 서술은 교과서 수준 중립 서술).
- **국어 2단원(각 300문)**: 맞춤법·띄어쓰기·높임법·문장성분 / 관용어·속담 뜻·쓰임.
- **전수 검증 파이프라인**(`scripts/bank_validate.ts`, CI성 스크립트):
  ① 스키마 검증 ② 수학 계산형 정답 재계산 일치 ③ 문항 텍스트 중복/유사도 탐지
  ④ correctIndex 분포 균형(각 25%±5) ⑤ 보기 중복 검사. **6,000문 all pass가 완료 조건.**

### 1.4 출제 로직

- 게임 시작 시 셔플, 진행 인덱스로 **한 판 무중복** 소진. 다 쓰면 "직전 20문 제외" 재셔플.
- 난이도 페이싱: 웨이브 1-3 → difficulty 1 위주, 4-8 → 2 혼합, 9+ → 3 가중.
- 타이머: calc/spelling 15초, word/fact/order 20초 (type별).

### 1.5 선택 UI 개편

- 흐름: **학년(5/6) → 과목(수학/사회/국어) → 단원 → 닉네임**.
- 단원 딥링크 `#/play?unit=<id>` 지원 — issamGPT에 단원별 개별 링크 제출 대응.

## 2. 에셋 제작 파이프라인 — "키비주얼 품질" 게이트 ★v1.1 신설

목표: 캐릭터/몬스터/무기/이펙트 **전 에셋이 키비주얼(B1·B2·B3)과 같은 화면에 놓여도
이질감이 없는 품질**을 시스템으로 보장한다. 개별 생성 운에 맡기지 않는다.

### 2.1 스타일 앵커링 (일관성의 핵심)

- **스타일 바이블 = 키비주얼 3장**: `docs/keyvisuals/B1(전투)·B2(라인업)·B3(타일)`.
- 모든 에셋은 **image-to-image**(`gpt-image-2-image-to-image`)로 생성하고, 앵커 이미지를
  `input_urls`에 항상 포함 → "이 스타일로 그려라"를 프롬프트가 아니라 이미지로 강제.
- 앵커 공개 URL 확보: kie.ai **File Upload API**(`POST /api/file-base64-upload` →
  `downloadUrl`, 3일 유지)로 배치 시작 시 자동 업로드. 외부 호스팅/레포 오염 없음.
- 신규 확정 에셋(예: 최종 히어로)은 앵커 세트에 추가되어 이후 생성물이 히어로와도
  일관되도록 체인. (`input_urls` 최대 16장)
- 공통 STYLE_CLAUSE(모든 프롬프트에 삽입): "16-bit SNES-era pixel art, crisp uniform
  pixel grid, 1px dark outline, top-left light source, 3/4 top-down view, limited palette
  matching the reference images, solid magenta background #FF00FF, single subject centered,
  no letters no words no numbers".

### 2.2 정량 품질 기준 (자동 검사 가능하게 수치화)

| 항목 | 기준 |
|------|------|
| 도트 밀도(native px) | 히어로 64 / 소형 몬스터 48 / 중형 64 / 보스 128 / 무기 32~48 / 이펙트 프레임 64 / 카드 아이콘 96 / 데코 48~96 |
| 마스터 팔레트 | B1+B2에서 추출한 ≤48색. 전 에셋 양자화 후 팔레트 밖 색 비율 0% |
| 외곽선 | 1px 다크 아웃라인 존재 (엣지 검출로 확인) |
| 투명도 | 배경 마젠타 잔여 픽셀 0, 반투명 fringe ≤1px |
| 프레임 정렬 | 시트 내 프레임 간 앵커(발바닥 baseline) 편차 ≤2px |
| IP 안전 | 슬라임 등 실루엣이 기존 IP(드래곤퀘스트 눈물방울형 등)와 불일치 — 육안 체크리스트 |

### 2.3 생성 규칙

- **애니메이션은 반드시 한 장의 가로 스트립(1×N)으로 생성** 후 슬라이스 — 프레임 간
  캐릭터 동일성 확보 (B2가 5캐릭터 균등 배치를 이미 입증). 프레임별 개별 생성 금지.
- **몬스터 15종은 패밀리 시트로 생성**: 슬라임계/비행계/야수계/정예계 4패밀리 × 4종을
  한 장씩 → 세트 내 일관성 + 상호 구분. 각 몬스터의 walk 2f는 후속 i2i로 스트립화.
- 보스 5종은 개별 생성(128px급 디테일), 공격 텔레그래프 1프레임 포함.
- 무기: 인게임 스프라이트(측면, 투사체용) 1장 + 카드 아이콘(정면 데코, 96px) 1장씩.

### 2.4 후처리 자동화 (`scripts/asset_pipeline.py` — gen_keyvisuals.py 확장)

```
generate(i2i, 앵커+프롬프트) → chroma-key 마젠타 제거 → 시트 프레임 분할(균등 셀)
→ 프레임 bbox 감지·발바닥 baseline 정렬 → native 해상도로 격자 스냅 다운스케일
  (셀 중앙값 샘플링) → 마스터 팔레트 양자화 → 알파 트림+2px 패딩
→ 아틀라스 패킹(JSON hash) + 애니메이션 매니페스트 갱신
```

- 중간 산출물은 `assets_work/`에 보존(디버깅), 최종본만 `public/assets/`.
- 현행 "1024² PNG를 0.07~0.2 스케일로 렌더" 낭비 제거 — native 크기로 탑재.

### 2.5 QA 게이트 (통과해야 게임에 들어감)

1. **자동 검사**: §2.2 기준 전 항목 스크립트 검증 (`scripts/asset_qa.py`).
2. **콘택트시트 육안 승인**: 배치별 `docs/asset_review/<batch>.html` 자동 생성 —
   전 에셋을 1x/2x/3x + 인게임 배경(B3 타일) 위에 배치 + 키비주얼과 나란히 표시.
   **사용자 승인 후에만 병합.**
3. **반려 루프**: 불합격 에셋은 프롬프트 보정 후 재생성(최대 3회, 비용 무관 정책).
   3회 실패 시 유사 합격 에셋을 i2i 소스로 파생 생성.

### 2.6 에셋 목록·수량

| 분류 | 내용 | 생성물 |
|------|------|--------|
| 히어로 | idle 2f + walk 4f + hurt 1f + death 4f (11프레임) | 스트립 2~3장 |
| 몬스터 | 15종 × walk 2f | 패밀리 시트 4장 + 스트립 15장 |

## 3. 애니메이션·모션 설계 ★v1.1 확장

### 3.1 캐릭터 상태머신 (Phaser anims — 현재 미사용 → 도입)

- `Player.ts`에 `AnimState = idle | walk | hurt | dead` + update에서 전이:
  velocity>0 → walk, 피격 이벤트 → hurt(120ms 후 복귀), HP 0 → dead.
- 클립 스펙: idle 2f@4fps 핑퐁 / walk 4f@10fps 루프(좌우는 flipX) /
  hurt 1f+화이트 tintFill / death 4f@6fps → 페이드 → 게임오버 오버레이.
- 애니메이션 등록은 BootScene에서 **매니페스트 JSON**(`assets/anims.json`) 일괄 등록 —
  에셋 파이프라인이 자동 생성하는 파일. 코드에 프레임 하드코딩 금지.
- **코드 마이크로 모션**(프레임 불필요, 즉시 체감):
  - walk bob: y ±1px sine (프레임레이트 동기)
  - 발밑 그림자 블롭: 별도 스프라이트, bob과 반대로 미세 스케일
  - 이동 시작/정지 squash-stretch: scale 1.0→0.92→1.0 (120ms)
  - 이동 중 4프레임마다 먼지 퍼프 파티클
  - 발사 반동: 조준 반대 방향 2px nudge (연사 무기는 버스트당 1회)

### 3.2 몬스터 모션

- 공용: walk 2f@6fps(스트립) + 코드 wobble(회전 ±4° sine, 개체별 위상차) →
  물량이어도 살아 움직이는 느낌.
- 스폰: 그림자 먼저 → 팝업 스케일(0→1.1→1.0, 150ms). 화면 밖 급증식 스폰 티 제거.
- 피격: 화이트 플래시(tintFill 80ms) + 넉백(기존 유지) + hit_spark.
- 사망: death_poof 시트 + 스케일 팝 + XP젬 방출 트윈(포물선 낙하).
- 보스: 등장 배너 + 카메라 셰이크, 공격 전 텔레그래프 프레임(빨간 아웃라인 펄스 500ms),
  피격 히트스톱 30ms, 처치 시 200ms 슬로모(time.timeScale).

### 3.3 무기 20종 모션 명세 (아키타입 8종 → 전 무기 매핑)

모두 **스프라이트 1장 + 코드 트윈/파티클** 중심(시트 최소화 → 제작 리스크 낮음).

| 무기 | 아키타입 | 모션 구현(코드) | 임팩트 |
|------|----------|-----------------|--------|
| 연필(시작무기) | 직선 관통 | 속도 방향 고정 회전, 속도선 트레일 | hit_spark 소 |
| 구슬 | 직선 반사 | 최대 사거리서 방향 반전 1회, 반짝 트윈 | hit_spark 소 |
| 바나나 | 회전 투척 | 부메랑 왕복(ease out-in), 자체 회전 0.15rad/f | hit_spark 소 |
| 도토리 | 바운스 | 낙하-튕김 3회, 튕길 때 squash(y 0.8) | 먼지 퍼프 |
| 별 | 낙하 강타 | 화면 상단서 랜덤 낙하, 착지 플래시+별가루 | hit_spark 대 |
| 종이비행기 | 유도 활공 | 사인파 궤적 + 뱅킹 롤(회전 진동), 유도 조향 | hit_spark 소 |
| 나비 | 유도 동료 | 날갯짓 2f@12fps + 사인 호버, 완만 유도 | heal 색 파티클 |
| 자(ruler) | 근접 호 | 90° 스윙 arc + 잔상 고스트 2장(알파 감쇠) | hit_spark 소 연쇄 |
| 크레파스 | 궤적 존 | 무지개 선 데칼이 궤적에 잔존(2s, 지속 데미지) | 색 파편 |
| 지우개 | 스탬프 | 낙하 스케일 펀치 + 하얀 "지운 자국" 데칼 페이드 + 지우개 가루 파티클 | 전용 |
| 도시락 | 로브 폭발 | 포물선(스케일 업→다운) + 텀블링 회전 → explosion 6f + 음식 파편 | explosion |
| 물풍선 | 로브 스플래시 | 비행 중 wobble 스케일 → splash 4f + 물방울 파티클 + 슬로우 링 | splash |
| 눈덩이 | 직선 성장 | 비행 중 스케일 1.0→1.3, 명중 시 슬로우 디버프 표식(눈꽃 아이콘 1s) | splash 백색 |
| 비눗방울 | 궤도 | 플레이어 궤도 회전 + 크기 숨쉬기(±8%), 터질 때 shine 링 | 전용 팝 |
| 햄스터 | 고속 궤도 | 궤도 회전 + 자체 스핀, 3s마다 바깥 대시 왕복 | hit_spark 소 |
| 로봇 장난감 | 동료 사수 | 옆에서 보행 2f, 1.5s마다 미니 볼트 발사(머즐 플래시 1f) | hit_spark 소 |
| 무지개 | 파동 | 반원 파동 스프라이트 확장(scale+알파 감쇠) + 색상 시프트(tint cycle) | 색 파편 |
| 돋보기 | 집광 빔 | 텔레그래프 원(축소 0.5s) → burn 4f + 연기 파티클, 지면 그을음 데칼 | burn |
| 자석 | 흡인 존 | 필드 파티클이 중심으로 흐름, 적에게 끌림+스트레치(x 1.15) | hit_spark 소 |
| 나뭇잎 | 표류 | 낙엽 사인 강하 + 회전 ±30°, 완만 유도 | 초록 파편 |

- 공통 규칙: 투사체 스폰 시 스케일 팝(0.7→1.0, 80ms), 소멸 시 알파 페이드 or 임팩트로 종결
  (허공에서 사라지는 투사체 금지).
- `WeaponBase`에 `spawnProjectile(cfg)` / `playImpact(x, y, kind)` 공통 헬퍼 추가 —
  현행 무기별 중복 코드(setInterval식 회전 등) 제거, 씬 tween/파티클로 통일.

### 3.4 공용 이펙트 라이브러리

- `EffectManager` + 오브젝트 풀: 파티클 이미터·임팩트 스프라이트 사전 생성/재사용
  (크롬북 GC 스파이크 방지). API: `fx.hit(x,y,size)`, `fx.poof(x,y)`, `fx.burst(kind,x,y)`.
- 시트 9종(§2.6)은 매니페스트로 자동 등록. 데미지 숫자(기존)는 크리티컬 시 확대+노랑.

## 4. 배경 시스템 — 이음매 근본 제거

## 5. 무기/패시브/선택지 로직

### 5.1 패시브 전면 교체 (VS 복붙 청산)

- 현행 20종 = Vampire Survivors 그대로(시금치/푸마롤라/메탈리오/티라기수…). 이 중
  **7종은 미구현 no-op**(greed, curse, crit×2, metaglio×2, tiragisu) — 함정 선택지.
- → **어린이 테마 12종으로 재구성** (예시안):
  우유(데미지)·책가방(방어)·비타민(최대HP)·반창고(재생)·알람시계(쿨다운)·돋보기안경(범위)·
  새총고무줄(투사체속도)·운동화(이동속도)·자석필통(수집범위)·네잎클로버(행운)·
  금별스티커(경험치)·응원부적(부활 1회, **실제 구현**).
- crit 시스템 신규 구현(행운 연동), greed/curse 폐기. `Player.applyStat` 스탯 전수 매핑 검증 테스트 추가.

### 5.2 무기 정리

- 20종 전원 자기 스프라이트 사용 (현재 8종만 사용, 11종은 단색 도형 폴백).
- `ruler` 정체성 복구: 시작 무기 = **연필(직선 관통)**로 교체, ruler는 "자 휘두르기(근접 호)"로
  재구현, 바나나 중복 제거.

### 5.3 선택지 제공 로직

- 슬롯 규칙 일원화: 무기 6/패시브 6. **슬롯 가득이면 신규 제안 제외**
  (현행 maxWeapons=6 vs maxWeaponChoices=8 불일치로 보상 증발 버그).
- 가중 추첨: 보유 강화 55% / 신규 무기 30% / 패시브 15%, 행운 보정(+신규·고급 확률).
- 제안 불가 상황(전부 만렙)엔 대체 카드: 체력 회복 30% / 점수 +500 / 자석 발동.
- 카드에 **다음 레벨 수치 명시**("데미지 +3", "투사체 +1") + 아이콘 + 레벨 점.

## 6. 퀴즈 루프 리디자인 (페널티 완화)

실플레이 검증된 문제: 오답/타임아웃 → 레벨 회수 + XP 전액 몰수 + 재개 직후 포위 즉사.

- 정답: 업그레이드 3택 + 점수 보너스 + **콤보 스트릭**(연속 정답당 XP 배율 +5%, 최대 +25%).
- 오답/타임아웃: **레벨업 소모(업그레이드 없음)로 단순화** — XP 몰수·레벨 회수 폐지.
- 해설: 오답 시 "확인" 버튼 누를 때까지 유지(읽기 시간 보장), 정답 시 1.2s 자동 진행.
- **재개 보호**: 1.5초 무적 + 반경 200px 몬스터 밀쳐내기 + "3·2·1" 카운트다운.
- `xpToNextLevel` 초기값 10 vs 리셋 25 불일치 수정(25 통일), XP 곡선 재조정
  (목표: 첫 5분에 퀴즈 4~5회).
